# 前后端API不匹配问题清单

## 严重问题（阻塞功能）

### 1. 缺失的详情查询API
**影响：** 前端无法查询单个资源的详细信息

| 前端调用 | 后端状态 | 优先级 |
|---------|---------|--------|
| `GET /api/v1/model/:id` | ❌ 未实现 | P0 |
| `GET /api/v1/apikey/:id` | ❌ 未实现 | P0 |
| `GET /api/v1/template/:id` | ❌ 未实现 | P0 |

**解决方案：** 需要在后端添加这三个端点

---

### 2. 使用统计API完全不匹配
**影响：** 前端统计功能无法使用

| 前端期望 | 后端提供 | 问题 |
|---------|---------|------|
| `GET /api/v1/statistics` | `GET /api/v1/cost/stats` | 路径不同 |
| 使用统计（调用次数、token、成功率） | 成本统计（费用、预警） | 功能不同 |

**解决方案：** 
- 选项A：后端添加 `/api/v1/statistics` 端点，提供使用统计
- 选项B：前端改用 `/api/v1/cost/stats`，调整统计展示逻辑
- 选项C：合并两个功能，提供统一的统计API

---

### 3. 健康检查API不匹配
**影响：** 前端健康检查功能无法使用

| 前端调用 | 后端状态 | 问题 |
|---------|---------|------|
| `GET /api/v1/health-checks` | ❌ 未实现 | 获取历史记录 |
| `POST /api/v1/model/:id/health-check` | ❌ 未实现 | 触发检查 |
| - | `GET /api/v1/health` ✓ | 仅当前状态 |

**解决方案：** 
- 后端添加健康检查历史记录API
- 后端添加触发健康检查API

---

## 中等问题（影响体验）

### 4. API密钥字段不匹配
**影响：** 前后端数据结构不一致，可能导致数据丢失

| 字段 | 前端 | 后端 | 问题 |
|------|------|------|------|
| 模型关联 | `model_config_id` | `model_id` | 字段名不同 |
| 提供商 | `provider` | - | 后端缺失 |
| 描述 | - | `description` | 前端缺失 |

**解决方案：** 统一字段命名，建议使用 `model_id`

---

### 5. 时间字段命名不一致
**影响：** 前端解析时间字段可能失败

| 资源类型 | 前端期望 | 后端返回 |
|---------|---------|---------|
| API Key | `created_time`, `updated_time` | `created_at`, `updated_at` |
| Template | `created_time`, `updated_time` | `created_at`, `updated_at` |
| Model | `created_at`, `updated_at` | 未确认 |

**解决方案：** 统一使用 `created_at` 和 `updated_at`（符合Go惯例）

---
