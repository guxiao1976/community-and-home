## Context

`md_administrative_division` 表是五级行政区划层级表（省→市→区县→乡镇→村/社区），当前表无数据，需要从国家统计局2022年统计用区划代码Excel文件批量导入。数据源位于 `/mnt/d/wsl/data/行政区划/`，共31个省级Excel文件，每个文件约7000+行五级数据。

Excel数据结构（每个文件）：
- Row 1: 表头（省级区划代码, 省级名称, 市级区划代码, 市级名称, 区县级区划代码, 区县名称, 乡镇级区划代码, 乡镇街道名称, 村级区划代码, 城乡分类代码, 村(居委)名称）
- Row 2: 各级统计数量（不是实际数据行，需跳过）
- Row 3+: 实际数据（每行包含完整的五级区划）

数据库: `root:123456@tcp(localhost:3306)/masterdata_db`

## Goals / Non-Goals

**Goals:**
- 清空现有数据并导入全国31省级行政区划五级数据
- 新增 `urban_rural_code` 列存储城乡分类代码
- 统一设置 `submission_status=2`, `status=1`
- 每个文件导入后数量校验并输出报告

**Non-Goals:**
- 不做增量更新或去重合并（直接全量替换）
- 不处理历史数据迁移
- 不做前端UI开发
- 不处理 `~$` 开头的临时文件

## Decisions

### 1. 使用Python脚本而非Go程序
**选择**: Python + openpyxl + pymysql
**理由**: openpyxl已安装可解析xlsx，pymysql已安装可直连MySQL；Python处理Excel和数据转换更灵活；这是一次性导入工具，不需要加入Go服务代码。
**替代方案**: Go程序 — 需要额外引入xlsx解析库，开发周期更长，不适合一次性脚本。

### 2. 新增 `urban_rural_code` 列
**选择**: `ALTER TABLE ADD COLUMN urban_rural_code VARCHAR(20) NULL` AFTER `sort_order`
**理由**: 城乡分类代码仅在第五级（村/社区）有值，其他级别为NULL。放在 `sort_order` 之后逻辑上与行政区划属性字段相邻。

### 3. 数据导入策略：去重合并跨文件数据
**选择**: 使用 `INSERT IGNORE` 基于 `code` 的唯一索引去重
**理由**: 五级数据在不同行中会有重复的上级区划（如"北京市"在7000+行中都出现），需要按层级去重后插入。同一code在多个文件中可能重复出现（理论上不应，但防御性编程）。

### 4. 数据导入方式：按层级分批插入
**选择**: 从每个文件提取五级数据后，按level=1→2→3→4→5顺序插入，先插入上级再插入下级
**理由**: 需要先获取parent_id来构建path。按层级顺序插入确保父记录已存在。

### 5. parent_id和path的构建
**选择**: 维护一个 `code→id` 的内存映射表，逐级构建 parent_id 和 path（如 `/省id/市id/区县id/乡镇id/`）
**理由**: 所有数据在一个事务中处理，内存映射效率最高，无需多次查询数据库。

## Risks / Trade-offs

- **数据量大（60万+条）** → 使用批量INSERT（每批1000条）和事务减少IO；预计总耗时2-5分钟
- **Excel临时文件（`~$` 开头）** → 脚本中跳过 `~$` 前缀文件
- **北京市文件存在两个版本** → 北京市1.xlsx 会被自动跳过（与北京市.xlsx数据重复），或需确认是否需要特殊处理
- **code唯一冲突** → 使用 `INSERT IGNORE` 静默跳过重复，在校验报告中记录
- **第12列(None)的存在** → 忽略该列，它不包含有用数据
